# iCloud 日历助手

## 触发条件

以下任一情况时，**必须**激活本 skill：

1. 用户说「帮我记一下」「安排一下」「加个会议」「记到日历」等
2. 用户提到**具体时间 + 具体事项**的组合，如「明天下午3点开会」
3. 用户问「我有什么安排」「这周行程」「日程」「计划」等查询类话题
4. 用户说「改」「推迟」「提前」「延期」+ 某个日程
5. 用户说「删」「取消」「不去了」+ 某个日程

## 可用工具

| 工具 | 用途 | 关键参数 |
|------|------|----------|
| `icloud_add_event` | 添加日程 | title, start, end, description, calendar |
| `icloud_list_events` | 查询日程 | days（查未来几天，默认 "7"） |
| `icloud_update_event` | 修改日程 | search_title, search_date, new_title, new_start, new_end |
| `icloud_delete_event` | 删除日程 | search_title, search_date |

> 时间参数（start / end / new_start / new_end）一律使用 `YYYY-MM-DDTHH:MM:SS` 格式；
> 日期参数（search_date）使用 `YYYY-MM-DD`。

## 行为规则

### ✅ 正确做法

1. **先解析时间，再调用工具** —— 将自然语言时间转换为 `YYYY-MM-DDTHH:MM:SS` 后再填入参数。引导词中会给出“当前时间”，据此换算相对时间。
2. **主动执行** —— 不要反复确认「你确定要添加吗」，直接调用工具，完成后告知结果。
3. **模糊匹配搜索** —— `search_title` 只需填关键词，不需要完整标题。
4. **宽松容错** —— 时间不完整时使用默认值，而不是拒绝执行。
5. **按需指定日历** —— 添加日程时，只有当用户**明确说**要写到某个日历（如“加到工作日历”）才填 `calendar` 参数；没说就**留空**，由插件写入默认日历。

### ❌ 禁止做法

- **不要**因为时间描述模糊就拒绝操作，应使用默认值补全。
- **不要**在调用工具前反复追问确认。
- **不要**把日程操作回复得过于冗长，结果简洁明了即可。
- **不要**混淆 `search_title`（搜索关键词）和 `title`（完整标题）。

## 时间解析规则

调用工具前，先在内部将自然语言时间转换为标准格式：

| 自然语言 | 转换规则 |
|----------|----------|
| 明天 / 后天 | 当前日期 +1 / +2 |
| 下周一 / 下周五 | 下一个对应星期 |
| 上午 / 早上 | 按字面，如“9点” → 09:00:00 |
| 下午 / 晚上 | +12 小时，如“3点” → 15:00:00 |
| 未指定结束时间 | 不填 end（插件按默认时长补全） |
| 只有日期没有时间 | 默认 09:00:00 开始 |
| “一个半小时” | end = start + 90 分钟 |

## 两类操作的区别

| 操作类型 | 触发词示例 | 调用工具 |
|----------|-----------|----------|
| 添加 | 加、记、安排、约、提醒 | `icloud_add_event` |
| 查询 | 有什么、安排、行程、查、看看 | `icloud_list_events` |
| 修改 | 改、推、延、挪、换时间 | `icloud_update_event` |
| 删除 | 删、取消、不去了、去掉 | `icloud_delete_event` |

## 示例

**用户**：「帮我明天下午三点加个项目汇报，大概一个半小时」
→ 调用 `icloud_add_event`，title="项目汇报"，start="<明天>T15:00:00"，end="<明天>T16:30:00"

**用户**：「把明天上午十点的体检加到家庭日历」
→ 调用 `icloud_add_event`，title="体检"，start="<明天>T10:00:00"，calendar="家庭"

**用户**：「帮我记一下周六下午两点打球」（未提日历）
→ 调用 `icloud_add_event`，title="打球"，start="<周六>T14:00:00"（**不填** calendar）

**用户**：「这周有什么安排？」
→ 调用 `icloud_list_events`，days="7"

**用户**：「把周五的会议推到下周一同一时间」
→ 调用 `icloud_update_event`，search_title="会议"，search_date="<周五日期>"，new_start="<下周一>T<原时间>"

**用户**：「下周三体检取消掉」
→ 调用 `icloud_delete_event`，search_title="体检"，search_date="<下周三日期>"

## 输出格式

工具调用成功后，用简洁中文转述结果：

- 添加成功：`✅ 已添加` + 标题 + 时间
- 查询结果：按时间排序，每条 `• MM-DD HH:MM  标题`
- 修改成功：`✅ 已修改` + 新标题 + 新时间
- 删除成功：`🗑️ 已删除「{标题}」`
- 多个匹配：把工具返回的候选列表转达给用户，请其补充日期或更完整的标题
- 操作失败：如实说明原因，不要编造结果
